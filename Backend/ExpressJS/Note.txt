Express.js :
--Express.js is a light weight Node.js framework design to make the development of server side applications straight forward by providing inbuilt methods for routing, handling http request/response and intigrating middleware easily
--Express.js is a minimalist web application framework built on Node.js which is primarily use to simplify the process of creating web servers.
--Express.js is flexible and fast which enables developers to built both small and large applications.
--It is commonly used to create APIs, web application and handling various http methods.

Use of Express.js :
--Built Web App and API using routing and middleware
--Reduces boilerplate code and coding time compare to VANILA Node.js
--Organises functionality with middleware for tasks like authentication, login, and errorhandling
--It allows rapid development of scalable and maintainable web application

Differences between Express.js and Node.js
--Routing :(Node.js) -> In Node.js manual setup mode code require
           (Express.js) -> Clean and builtIn routing system

--JSON Parsing :(Node.js) -> manual or 3rd party
                (Express.js) -> simple by using Express.json()

--Middleware :(Node.js) -> needs 3rd party library
              (Express.js)-> BuiltIn and systematic middleware support

Understanding request and response in Express.js :-
request: It represents the http request and contains data sent by the client including path parameters, query strings body data, headers and more. 
response : It represents http response that an express app sends back to the client, it provides methods to send data like 
     res.send(), res.json(), res.status()......etc 

app.get("/",(req,res)=>{
    res.send("This is an example of response")
    console.log(req.method)
})     

Routes in Express.js:-
--Routing defines how an applications end point respond to client requset 
--Express.js uses method like get(), post(), put(), delete() etc to define routes according to http methods.
--Routes consist of a path and a callback func with req and res parameters.
--There are 2 types of routes :  normal route and parameterized route 
 for ex:-  
app.get("/books",(req,res)=>{
     res.send("normal route")
}) 
app.get("/books/:id",(req,res)=>{
    res.send("parameterized route")   
})

Route Parameter:
--Route parameters are dynamic segments in the url path defined by using a single column(/books/:id),it allows capturing value directly from the url which are than accecible by using req.params in express
  for Ex:
  app.get("/books/:id",(req,res)=>{
    res.send(parameterized route ${req.params.id})   
  }) 

--In express multiple parameters are allowed.
--Route parameters are used to select a specific resource like fetching a specific user by id.


QUERY STRING
--Query strings are key value pairs appended to the end of a url after a queustion mark (?)
  e.g: ?price=10000&color=red
--In express query string are accessed in express via req.query object
  e.g: app.get('/search',(req,res)=>{
     res.send(Search for: ${req.query.q})
})

--Query string are used to filtering data ,searching or providing additional parameters to a request.

Route Chaining: 
--Route chaining allows defining multiple http methods for a single route using app.rout()
--It keeps route declarations organized especially when handling different operation on one path. 
e.g app.route('/books')
    .get((req,res)=>{...})
    .post((req,res)=>{...})
    .put((req,res)=>{...})
    .delete((req,res)=>{...})

Q.1 What is the role of a middleware?
 It is handle req and res  between the user and server.

#Middleware:-
-It is a function or software layer that sits between the request and response cycle in web applications in framework like Express.js which enable developer to add ,modify and handle functionalities for incoming request before they reach the final route handler.
-It acts as an intermediate between frontend and backend for communication ,requesting ,handling,logging ,security and more.
-In simpe we can say it is a function that receives request, response and the next callback as an argument which allows executing codes and make changes to request and request respond object.

//Use of Middleware:
-Middleware is wideely used for tasks such as authentication, login, error handling, passing request body and adding custom headers.
-Some common built in and third party Middleware are expess.json(),customlogger(), cors().
-Middleware can be applied globally by using app.use() or to specific route by using app.get()/ app.post().

#What is Next()?
-The next() in Middleware passes control to the other Middleware function in the stack.
-If a Middleware does not end the request respond cycle  it must call th enext() otherwise the request will hang and not be processed futher.

#Types of Middleware:-
1)Build in Middleware
2)Third-party Middleware
3)Custom Middleware
4)Application-level Middleware
5)Router-level Middleware

#EXPRESS ROUTER:-
-It is a feature in express.js that allows us to create modular and mountable route handlers acting like a mini express application for specific part of API.
-It helps us to organised routes into separates files like /users, /products, /login so large app stay clean, reusable and easy to maintain.
-Express.router() create a separate router project or variable that can have its own routes and Middleware similar to the main app.

#STATIC FILES IN EXPRESS:-
-Express.js provide a built-in Middleware express.static() to serve static file from a folder such as public or assets.
-Static file in express are files like image, CSS, client side JavaScript, pdf, etc that are sent to the browser without any server-side processing.
-By using static() any file inside public folder becomes directly accessible by its path.
-We can mount multiple static folders or add a url prefix like app.use('/assets',express.static('public')) to serve file under a specific route segment.
